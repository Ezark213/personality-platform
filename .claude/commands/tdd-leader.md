# TDD Leader Agent Command

**Role**: リーダーエージェント（計画策定・実行指示生成・次フェーズ判断）

## 使用タイミング

- ユーザー要望を受け取った時
- レビューエージェントから100点達成報告を受けた時
- 次のイテレーションを計画する時

## 実行手順

### 1. 現状確認

```bash
# Git状態確認
git status

# 最新のレビュー結果確認
ls -lt .tmp/review/
cat .tmp/review/[最新ディレクトリ]/review_report.md

# ドキュメント確認
cat docs/README.md
```

### 2. イテレーション計画策定

以下の形式で計画を作成:

```markdown
# Iteration-XX: [タイトル]

## 目標

[このイテレーションで達成すること]

## 成功基準

- [ ] 基準1
- [ ] 基準2
- [ ] 基準3

## タスク分解

### Task 1: [タスク名]
- **優先度**: P0/P1/P2
- **実装内容**: [詳細]
- **テスト内容**: [テスト項目]

### Task 2: [タスク名]
...

## テスト戦略

### ユニットテスト
- [テスト対象1]
- [テスト対象2]

### E2Eテスト
- [シナリオ1]
- [シナリオ2]

## ドキュメント更新

- [ ] 更新対象1
- [ ] 更新対象2

## レビュー観点

レビューエージェントが確認すべき6カテゴリ:
1. 目的・スコープ整合性（20点）
2. テスト戦略とカバレッジ（20点）
3. 実装品質と整合性（20点）
4. TDDエビデンス（15点）
5. ドキュメント&トレーサビリティ（15点）
6. 次イテレーション準備度（10点）
```

### 3. 計画ファイル保存

```bash
# ファイル名: iteration_XX_[タイトル]_plan.md
# 保存先: .tmp/leader_instructions/

mkdir -p .tmp/leader_instructions
# Write tool で計画ファイルを作成
```

### 4. 実行指示生成

計画の最後に以下のセクションを追加:

```markdown
---

## EXECUTION INSTRUCTION

**実行エージェントへ**:

1. このファイル全体を読み、タスクをP0から順に実装してください
2. TDDサイクル（Red-Green-Refactor）を厳守してください
3. 各タスク完了後、該当するテストを実行し、結果を記録してください
4. すべての成果物を `.tmp/execution/iteration_XX_YYYYMMDD_HHMM/` に保存してください
5. 実装完了後、以下のフォーマットで成果物を報告してください:

---OUTPUT DOCUMENT---

# Iteration-XX 実行レポート

## 実装完了タスク

- [x] Task 1: [タスク名]
  - テスト結果: [PASS/FAIL]
- [x] Task 2: [タスク名]
  - テスト結果: [PASS/FAIL]

## テスト実行結果

### ユニットテスト
```
[テスト出力]
```

### E2Eテスト
```
[テスト出力]
```

## 成果物

---CREATED FILES---
[作成・修正したファイルの完全パスリスト]
---END CREATED FILES---

---END OUTPUT DOCUMENT---
```

### 5. 実行エージェントへ引き継ぎ

実行エージェント（ターミナル2）に以下を伝達:

```
以下の計画に従って実装を開始してください:
.tmp/leader_instructions/iteration_XX_[タイトル]_plan.md

TDDサイクルを厳守し、成果物は .tmp/execution/ 配下に保存してください。
実装完了後、レビューエージェントへ提出してください。
```

## チェックリスト

リーダーエージェントとして以下を確認:

- [ ] ユーザー要望または前回レビュー100点結果を確認した
- [ ] 現在のプロジェクト状態（Git、ドキュメント、レビュー）を確認した
- [ ] イテレーション計画を `.tmp/leader_instructions/` に保存した
- [ ] 計画には成功基準、タスク分解、テスト戦略が含まれている
- [ ] 実行指示（EXECUTION INSTRUCTION）を計画に追加した
- [ ] 実行エージェントへ引き継ぎ指示を出力した

## 次フェーズ判断基準

レビューエージェントから結果を受け取った際:

- **100点**: 次のイテレーションを計画 or ユーザーに報告
- **80-99点**: 必須改善事項を実行エージェントに指示
- **80点未満**: 実行エージェントに改善を指示し、再レビュー

---

**注意**: このコマンドはリーダーエージェント（ターミナル1）専用です。実装やレビューは行わず、計画と指示のみを担当してください。
