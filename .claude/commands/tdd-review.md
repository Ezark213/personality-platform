# TDD Review Agent Command

**Role**: レビューエージェント（80点必達レビュー・改善指示）

## 使用タイミング

- 実行エージェントから成果物提出を受けた時
- 改善後の再レビュー時

## レビュー基準（100点満点）

### 1. 目的・スコープ整合性（20点）

- [ ] 計画で定義された成功基準を全て達成している（10点）
- [ ] タスク分解で指定された全てのタスクが完了している（5点）
- [ ] スコープ外の実装が含まれていない（5点）

**評価**:
- 18-20点: 完璧な整合性
- 15-17点: 軽微な逸脱あり
- 10-14点: 重大な逸脱あり
- 0-9点: スコープ外実装が多数

### 2. テスト戦略とカバレッジ（20点）

- [ ] 全ての関数にユニットテストが存在する（8点）
- [ ] E2Eテストが主要フローをカバーしている（7点）
- [ ] テストが実際に実行され、結果が記録されている（5点）

**評価**:
- 18-20点: 網羅的なテストカバレッジ
- 15-17点: 一部テスト不足
- 10-14点: テストが不十分
- 0-9点: テストがほとんどない

### 3. 実装品質と整合性（20点）

- [ ] TypeScriptエラー0件（5点）
- [ ] ESLint/Prettier準拠（3点）
- [ ] ビルド成功（5点）
- [ ] コードが計画の意図を正しく実装している（7点）

**評価**:
- 18-20点: 高品質な実装
- 15-17点: 軽微な品質問題
- 10-14点: 品質問題が複数
- 0-9点: 品質基準未達

### 4. TDDエビデンス（15点）

- [ ] Red-Green-Refactorサイクルの証跡がある（8点）
- [ ] テストファーストで実装されている（7点）

**評価**:
- 13-15点: TDDサイクル明確
- 10-12点: TDD証跡やや不足
- 5-9点: TDD証跡不明確
- 0-4点: TDDサイクル未実施

### 5. ドキュメント&トレーサビリティ（15点）

- [ ] 実行レポートが完全（OUTPUT DOCUMENT形式）（5点）
- [ ] CREATED FILESに全成果物の完全パスが記載されている（5点）
- [ ] コードコメントが適切（5点）

**評価**:
- 13-15点: 完璧なドキュメント
- 10-12点: 軽微な不足
- 5-9点: ドキュメント不足
- 0-4点: ドキュメントほぼなし

### 6. 次イテレーション準備度（10点）

- [ ] 技術的負債が最小限（3点）
- [ ] 拡張可能な設計（4点）
- [ ] 既知の問題が文書化されている（3点）

**評価**:
- 9-10点: 次フェーズに即移行可能
- 7-8点: 軽微な改善で移行可能
- 4-6点: 改善が必要
- 0-3点: 大幅な改善が必要

## レビュー実行手順

### 1. 成果物読み込み

```bash
# 実行レポート確認
cat .tmp/execution/iteration_XX_YYYYMMDD_HHMM/execution_report.md

# 成果物ファイル確認
ls -la .tmp/execution/iteration_XX_YYYYMMDD_HHMM/

# 実装コード確認
# Read tool で CREATED FILES に記載されたファイルを読む
```

### 2. 計画との照合

```bash
# 元の計画を確認
cat .tmp/leader_instructions/iteration_XX_[タイトル]_plan.md

# 成功基準を確認
# タスク分解を確認
```

### 3. テスト実行確認

実行レポートから以下を確認:

- ユニットテスト結果
- E2Eテスト結果
- ビルド結果
- TDDサイクル証跡（Red→Green→Refactor）

### 4. コード品質確認

CREATED FILESに記載された各ファイルをRead toolで読み込み、以下を確認:

- TypeScript型注釈の適切性
- any型の使用有無
- コードの可読性
- テストコードの品質

### 5. スコアリング

各カテゴリのスコアを計算し、総合スコアを算出:

```
総合スコア = カテゴリ1 + カテゴリ2 + ... + カテゴリ6
```

### 6. レビューレポート作成

以下の形式で保存:

```markdown

レビュー日時: YYYY-MM-DD HH:MM
レビューエージェント: Claude Sonnet 4.5
```

### 8. 結果伝達

実行エージェント（ターミナル2）に改善指示:

```

```

#### 80-99点の場合

実行エージェントに推奨改善を伝達:

```

改善推奨事項が X 件あります。

1. [推奨事項1]

```

#### 100点の場合

リーダーエージェント（ターミナル1）に報告:

```

レビューレポート: .tmp/review/iteration_XX_YYYYMMDD_HHMM_review/review_report.md

次のイテレーションに進んでください。
```

## チェックリスト

レビューエージェントとして以下を確認:

- [ ] 実行レポートを読み込んだ
- [ ] 元の計画と照合した
- [ ] 6カテゴリ全てをスコアリングした
- [ ] 総合スコアを計算した（XX/100点）
- [ ] 80点未満の場合、具体的な改善指示を記載した
- [ ] 80-99点の場合、推奨改善事項を記載した
- [ ] レビューレポートを `.tmp/review/` に保存した
- [ ] 適切なエージェント（実行 or リーダー）に結果を伝達した

## レビューのコツ

### 厳格にレビューすべき点

- TypeScriptエラー（0件必須）
- テストの実行結果（全てパス必須）
- 成功基準の達成（全て達成必須）
- TDDサイクルの証跡（Red→Green→Refactor）

### 柔軟に評価すべき点

- コードスタイル（ESLint/Prettier準拠であれば許容）
- コメントの詳しさ（過剰なコメントは不要）
- リファクタリングの程度（動作すれば十分）

---

**注意**: このコマンドはレビューエージェント（ターミナル3）専用です。計画や実装は行わず、レビューと改善指示のみを担当してください。
