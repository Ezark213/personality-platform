# Iteration-02 実装決定事項

## Day 1: タイプ分類システム

### 1. タイプ分類の設計

#### 15タイプの定義方針

**基本方針**: 5次元×3レベル = 15タイプ

| 次元 | Low | Neutral | High |
|------|-----|---------|------|
| Neuroticism | 揺るがない楽観主義者 | バランス型の調整者 | 感受性豊かな共感者 |
| Extraversion | 内省的な思考家 | 柔軟な交流者 | 社交的なリーダー |
| Openness | 実践的な専門家 | 現実的な革新者 | 創造的な探究者 |
| Agreeableness | 率直な挑戦者 | 現実的な協力者 | 温かい調和の創り手 |
| Conscientiousness | 自由な即興家 | 適応的な実行者 | 計画的な達成者 |

#### スコアレベルの閾値

```typescript
// 0-100のnormalizedスコアから判定
- low: 0-35
- neutral: 36-64
- high: 65-100
```

**決定理由**:
- 中央値50を中心に±15の範囲をneutralとする
- 30%（low）、40%（neutral）、30%（high）の分布を想定
- 統計的に妥当な分割（正規分布の仮定）

### 2. タイプ分類アルゴリズム

#### 基本ロジック

```typescript
1. 各次元のnormalizedスコア（0-100）から中立値（50）との偏差を計算
   deviation = |normalizedScore - 50|

2. 偏差が最も大きい次元を主次元として選択

3. 同点の場合、以下の優先順位で決定:
   extraversion > conscientiousness > openness > agreeableness > neuroticism

4. 主次元のlevel（high/neutral/low）と組み合わせてタイプを特定
```

#### 優先順位の根拠

| 順位 | 次元 | 理由 |
|------|------|------|
| 1 | Extraversion | 最も観察可能で一貫性が高い特性 |
| 2 | Conscientiousness | 人生成果の強力な予測因子 |
| 3 | Openness | 認知スタイルを反映、創造性に影響 |
| 4 | Agreeableness | 社会的相互作用に影響 |
| 5 | Neuroticism | 状況により変動しやすい |

**参考文献**:
- Judge, T. A., & Bono, J. E. (2001). Relationship of core self-evaluations traits with job satisfaction and job performance.
- Costa, P. T., & McCrae, R. R. (1992). Four ways five factors are basic.

### 3. 共有テキスト生成の設計

#### Twitter/X共有フォーマット

```
私の性格診断結果は「{タイプ名}」でした！

✅ {強み1}
✅ {強み2}
✅ {強み3}

確かに当たってる！
あなたの結果は？診断してみてください！

#性格診断 #自己理解 #BigFive
{結果ページURL}
```

#### 文字数制御ロジック

**目標**: 140-200字

**調整アルゴリズム**:
1. 全要素を含めた基本形を生成
2. 200字超過の場合:
   - リアクション文（「確かに当たってる！」）を削除
   - CTAを短縮（「あなたの結果は？診断してみてください！」→「あなたの結果は？」）
   - URLを削除
3. 140字未満の場合:
   - フィラーを追加（「性格診断で自分を知ろう！」など）
   - 複数のフィラーをループで追加

**決定理由**:
- Twitter/Xの推奨文字数は140-280字
- OGP画像と合わせて共有するため、テキストは簡潔に
- ハッシュタグは必須（バイラル性確保）

#### ハッシュタグ選定

- `#性格診断`: 日本で人気の一般的なハッシュタグ
- `#自己理解`: 自己啓発系の検索流入を狙う
- `#BigFive`: 心理学的根拠をアピール

### 4. データ構造の決定

#### BigFiveType インターフェース

```typescript
export interface BigFiveType {
  id: string                    // 'high-extraversion' など
  name: string                  // '社交的なリーダー' など
  catchphrase: string           // '人と共に成長する' など
  strengths: string[]           // 3つの強み
  description: string           // 詳細説明
  primaryDimension: BigFiveDimension
  level: 'high' | 'neutral' | 'low'
}
```

**決定理由**:
- シンプルで拡張可能
- OG画像生成に必要な情報を全て含む
- タイプIDで一意に識別可能

### 5. テスト戦略

#### カバレッジ目標

- **タイプ分類**: 全15タイプ + エッジケース
- **共有テキスト**: 文字数、コンテンツ、フォーマット、エッジケース

#### テストケース設計

1. **正常系**:
   - 各次元のhigh/low/neutral分類
   - 典型的なスコアパターン

2. **境界値**:
   - 同点時の優先順位
   - スコア境界（35, 65）

3. **エッジケース**:
   - 全次元同点
   - 空URL
   - 極端に長いタイプ名

### 6. 技術的制約と対応

#### TypeScript strict mode

**制約**: 全コードはTypeScript strictモードに準拠
**対応**:
- 全ての型を明示的に定義
- nullチェックを徹底
- anyの使用を禁止

#### ファイル構成

**決定**:
- データ: `data/tests/`
- ロジック: `lib/tests/`
- テスト: `lib/tests/__tests__/`

**理由**:
- 既存のプロジェクト構造に従う
- テストコードとの対応関係が明確

### 8. 既知の問題と制限事項

#### タイプ分類の単純さ

**問題**: 最も偏差が大きい1次元のみで分類
**制限**: 複数の次元が高い/低い場合の複雑な性格を表現できない
**将来の改善**: クラスタリング分析で複合タイプを定義

#### 共有テキストの決定論性

**問題**: ランダム要素がないため、同じ結果は常に同じテキスト
**制限**: バイラル性が低下する可能性
**将来の改善**: リアクション文をランダム選択（A/Bテストで効果測定）

#### 既存テストとの非互換性

**問題**: `bigfive-calculator.test.ts`の3テストが失敗
**原因**: Iteration-01の型定義と実装の不整合
**対応**: Iteration-02の範囲外、別途修正が必要

### 9. パフォーマンス考慮

#### 計算量

- **タイプ分類**: O(1)（5次元の固定ループ）
- **共有テキスト生成**: O(1)（文字列操作のみ）

#### メモリ使用量

- **タイプマスター**: 15タイプ × ~200B = 3KB
- **キャッシュ不要**: 計算コストが低いため

### 10. セキュリティ考慮

#### XSS対策

- **対策**: ユーザー入力はURL のみ（診断結果は内部生成）
- **検証**: URLエンコーディングを徹底

#### プライバシー

- **方針**: 診断結果はクライアント側のsessionStorageのみ保存（サーバー保存なし）
- **注意**: Iteration-05でDB保存時はGDPR対応が必要
